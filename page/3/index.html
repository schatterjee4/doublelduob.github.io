gerrit使用入门（配置）

Posted on 2015-07-17

目的

帮助要使用gerrit的用户，完成gerrit的基本配置，不会涉及gerrit是什么，为什么要codereview的部分，更详细的gerrit的使用，可以参考Learning Gerrit Code Review，配置部分为chapter4

准备

已经有可用的gerrit服务，并且有权限clone上面的某个项目

步骤

  • Mac／Linux用户跳过这一步，Win用户推荐先去下载gitbash,安装后，右键打开gitbash终端，不能下载的小伙伴请用 wad6到百度云提取1.9.5版本
  • 参考Git-生成-SSH-公钥生成SSH的public key，把~/.ssh/id_rsa.pub的内容拷贝出来，粘贴到gerrit个人配置的ssh选项中

  • 点击 Projects->List到 gerrit的项目列表（admin可以创建新项目）

  • 选择参与的项目，例子中是demo这个项目

  • 在Project ${name} 下面有5个不明显的按钮，建议选择clone with commit-msg hook和SSH这两项

  • 如果看不到这两项，可能是右上角用户名->settings里面还没有设置用户名

  • 拷贝那串很长的命令，或者点击命令后面的公文包按钮，粘贴到终端（win用户gitbash）中执行，看到如下结果，说明配置成功

推荐资料

内部gerrit和github用不同的ssh密钥，可以参考git多账号切换
